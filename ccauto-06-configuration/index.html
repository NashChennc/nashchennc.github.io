三个可定制 Slot

The classifier’s prompt is a fixed template we ship, with three customizable slots inserted into the middle. The fixed parts do the interpretive work… The slots define your policy: what counts as trusted in your environment, what categories to block, what exceptions to carve out.

— Claude Code auto mode

分类器的 system prompt 由固定模板和三个可替换 Slot 组成：

Slot 注入方式

buildYoloSystemPrompt（职责：将 Slot 规则注入固定分类模板；src/utils/permissions/yoloClassifier.ts L484-540）通过 XML 标签替换将用户规则注入模板：

替换语义：若用户提供了某 Slot 的内容，整段替换（REPLACE 而非追加）；若为空，保留模板默认值。这意味着用户的自定义规则会替换而非叠加默认规则。

Slot 内容来源

allow Slot 还会合并 Bash prompt allow descriptions（从 context 中的 Bash(prompt:...) 规则提取；参数 context 承载当前权限规则上下文），格式化为 - 每项一行。

soft_deny Slot 会合并：

  • Bash prompt deny descriptions
  • PowerShell 指引（如果适用）
  • 用户 autoMode.soft_deny 设置

environment Slot 仅来自用户 autoMode.environment 设置。

路径对照：userSettings → ~/.claude/settings.json；localSettings → .claude/local-settings.json；flagSettings → GrowthBook；policySettings → MDM / 远端策略。projectSettings（.claude/settings.json）不在 for 数组内，防恶意项目注入分类器规则（RCE）。

getAutoModeConfig（职责：聚合多来源 autoMode 配置并做安全过滤；src/utils/settings/settings.ts L936-982）的合并逻辑：

  1. 按固定顺序遍历：userSettings → localSettings → flagSettings → policySettings
  2. 故意不包含 projectSettings——防止恶意项目通过 .claude/settings.json 注入分类器规则
  3. 每个来源的 autoMode 字段做 zod 校验
  4. allow、soft_deny、environment 各自追加合并为一个大数组（多源累积）
  5. Ant 内部：autoMode.deny 会并入 soft_deny

相同的排除策略应用于其他安全设置

claude auto-mode critique

使用 AI（通过 sideQuery）审查用户自定义规则的质量：

将用户规则、默认规则、完整分类器 system prompt 一起交给模型，按四个维度评估：

  • Clarity：规则是否无歧义？
  • Completeness：是否有遗漏的边界情况？
  • Conflicts：规则之间是否冲突？
  • Actionability：规则是否具体到分类器可执行？

这个子命令展示了一个有趣的"meta"设计：用 AI 审查给 AI 分类器的规则。

Plan Mode 下的 Auto Mode

Auto mode 可以在 plan mode 期间保持活跃（getUseAutoModeDuringPlan，职责：读取是否在 plan 期间维持 auto 的设置；src/utils/settings/settings.ts L918）。相关逻辑由 transitionPlanAutoMode（职责：在 plan 入口/出口对齐 auto 状态；src/utils/permissions/permissionSetup.ts）处理：

  • 进入 plan 时：如果 shouldPlanUseAutoMode()，setAutoModeActive(true) + stripDangerousPermissionsForAutoMode
  • 退出 plan 时：ExitPlanModeV2Tool 检查 isAutoModeActive() 恢复 auto 状态
  • 设置变更时：Config.tsx 的 onChange 立即调用 transitionPlanAutoMode 对齐状态

设置界面中的 “Use auto mode during plan” 开关写入 userSettings，修改立即生效（不需要退出再进入 plan）。
