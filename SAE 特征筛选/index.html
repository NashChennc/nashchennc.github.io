SAE 特征筛选

2026 Apr 03
笔记
12 min read

## 平均激活差 (Difference in Means)。

1. 数学定义与几何解释

假设对于某个特定的 SAE 特征 $f_i$：

  • 正样本集 (Positive/Unsafe): $X_{pos} = {x_1, x_2, …, x_m}$

  • 负样本集 (Negative/Safe): $X_{neg} = {y_1, y_2, …, y_n}$

你所说的指标 $\Delta \mu$ 计算如下：

$$\Delta \mu = \frac{1}{m}\sum_{x \in X_{pos}} x - \frac{1}{n}\sum_{y \in X_{neg}} y = \mu_{pos} - \mu_{neg}$$

几何解释：

如果你把所有的样本看作高维空间中的点，这个指标计算的是两个类别中心点（Centroids）在特征 $f_i$ 这一维度上的投影距离。

  • 如果在所有维度上都做这个减法，得到的向量其实就是连接正样本中心和负样本中心的向量，这正是 Steering Vector (引导向量) 的基础形式。

2. 这个指标的性质 (Properties)

A. 具有方向性 (Signed / Directional)

与互信息（MI）或卡方（$\chi^2$）不同，$\Delta \mu$ 是有正负号的。

  • $\Delta \mu > 0$: 该特征在 Unsafe 样本中激活更强（Unsafe-correlated）。

  • $\Delta \mu < 0$: 该特征在 Safe 样本中激活更强（Safe-correlated）。

  • 价值： 这对于你后续构建“不同类别的森林”非常有用，你可以直接通过符号把特征分流给不同的专家模型。

B. 尺度敏感 (Scale Sensitive)

这是该指标最大的双刃剑。

  • 它直接依赖于激活值的绝对大小。

  • 如果特征 A 的激活值范围是 $[0, 100]$，特征 B 的范围是 $[0, 1]$。即使特征 B 分类能力更强（完美分割），特征 A 的 $\Delta \mu$ 可能因为数值本身很大而远大于 B。

  • 后果： 如果不进行归一化（Normalization），这个指标会偏向于那些“嗓门大”（激活值很高）的特征，而忽略那些“声音小但很准”的特征。

C. 线性假设 (Linearity Assumption)

它假设差异是累加的。它只关心“平均水平”的移动，而不关心分布的形状。

3. 评价：在 SAE 场景下的优缺点

✅ 优点 (Pros)

  1. Steering 意义明确:

    对于 SAE 来说，激活值的大小通常代表“语义强度”。$\Delta \mu$ 大，意味着在 Unsafe 样本中这个语义确实更强。这是构建 Steering Vector 最直接的方法。

  2. 计算极快:

    复杂度为 $O(N)$，且只需要做加减法，不需要像计算互信息那样做复杂的概率密度估计或分箱。

  3. 稀疏性友好 (部分):

    由于 SAE 大部分时候激活为 0，平均值的计算会自动把“激活频率”和“激活强度”结合在一起。

❌ 缺点与风险 (Cons & Risks)

  1. 忽略方差 (Ignores Variance / Overlap):

    这是最致命的缺陷。

    • 情况 A: Unsafe 均值 10，Safe 均值 0。方差都很小。$\Delta = 10$。（完美区分）

    • 情况 B: Unsafe 均值 100，Safe 均值 90。方差极大（比如 50），两类数据混在一起。$\Delta = 10$。（区分度极差）

    • 结论： 仅看差值，A 和 B 得分一样，但 A 明显更好。这个指标无法区分“分得开”和“仅仅是均值大”。

  2. 受离群点影响 (Outlier Sensitivity):

    SAE 特征有时候会因为某个特定的 token 突然爆出一个极大的激活值（例如 1000）。

    • 如果在 Safe 样本中只有一个样本激活了 1000，其他都是 0。平均值会被瞬间拉高，导致 $\Delta \mu$变成负数，让你误以为这是个 Safe 特征，其实它只是噪音。

  3. 混淆“频率”与“强度”:

    • 特征 X：在 100% 的 Unsafe 样本中以强度 1 激活。（$\mu_{pos} = 1$）

    • 特征 Y：在 1% 的 Unsafe 样本中以强度 100 激活。（$\mu_{pos} = 1$）

    • 两者均值相同，但物理含义完全不同。特征 X 是普遍特征，特征 Y 是特定场景特征（比如特定攻击词汇）。直接相减无法区分这两种情况。

4. 改进建议：标准化平均差

为了解决上述“忽略方差”的问题，统计学中通常使用 Cohen’s d 或 T-statistic 的思想来修正这个指标。

建议你不要只用 $\mu_{pos} - \mu_{neg}$，而是用：

$$Score = \frac{\mu_{pos} - \mu_{neg}}{\sigma_{pooled} + \epsilon}$$

  • $\sigma_{pooled}$：混合标准差（代表数据的离散程度）。

  • 含义： 这是一个信噪比 (Signal-to-Noise Ratio) 的概念。我们不仅要求正负样本均值差得远（Signal 大），还要求它们各自内部比较聚拢（Noise 小）。

总结

  • 正负激活相减是一个**一阶矩（First-order moment）**统计量。

  • 适用性： 它可以作为粗筛，或者用于确定特征的方向（是 Safe 还是 Unsafe）。

  • 局限性： 作为一个特征选择的“金标准”是不够的，因为它忽略了方差和分布重叠。

  • 建议： 如果计算资源允许，最好结合 方差归一化（变成 T-score）或者结合 互信息（捕捉非线性）一起使用。

1. 核心对比总表 (The Theoretical Matrix)

指标 (Metric) 统计阶数 (Order/Moment) 核心代数运算 (Algebra) 处理的信息 (Information Processed) 对分类问题的本质视角
均值差 (Mean Diff) 一阶 (1st Order) 减法 (Linear Subtraction) $\mu_1 - \mu_2$ 位置 (Location): 仅关注两个分布中心点的距离，忽略形状。 “这个特征在正负样本中的强度是否不同？”
相关系数 (Correlation) 二阶 (2nd Order) 内积/归一化 (Dot Product) $\frac{E[XY]}{\sigma_X \sigma_Y}$ **线性共变 (Linear Co-variance):**关注特征与标签是否同步变化。 “特征增加时，是 Safe 还是 Unsafe 的概率线性增加？”
Precision / Recall 基于阈值 (Threshold-based) 比率 (Ratio) $TP/(TP+FP)$ 象限计数 (Quadrant Counts): 关注混淆矩阵的特定部分，忽略 True Negatives (通常)。 “如果我根据这个特征判定为 Unsafe，判对的概率有多大？”
F1 Score 基于阈值 (Threshold-based) 调和平均 (Harmonic Mean) $\frac{2PR}{P+R}$ 平衡计数 (Balanced Counts): 寻求 P 和 R 的权衡点。 “这个特征在查全和查准之间是否有一个好的平衡点？”
卡方 ($\chi^2$) 二阶 (2nd Order) 二次离差 (Quadratic) $\sum \frac{(O-E)^2}{E}$ 独立性偏离 (Deviation from Independence): 关注实际分布与随机分布的距离。 “这个特征的出现频率是否显著异常于随机猜测？”
互信息 (Mutual Info) 全阶 (All Orders) 对数 (Logarithmic) $\sum p \log p$ 不确定性削减 (Entropy Reduction): 关注整个概率分布的形状重合度。 “已知这个特征后，我对标签的困惑度下降了多少比特？”

2. 深度剖析：从信息论与统计学角度

第一类：几何与线性指标 (均值差, 相关系数)

  • 统计学视角： 这是一阶（均值）或二阶（方差/协方差）矩的比较。它们假设数据服从单峰分布（如高斯分布），且关系是线性的。

  • 代数本质： 线性变换。

  • 信息损失： 极大。它们“压缩”了整个分布的形状信息。

    • 例子： 一个 SAE 特征在 Safe 类中是双峰分布（比如值在 0 和 10），在 Unsafe 类中是单峰（值在 5）。均值可能都是 5，均值差为 0。这个指标认为该特征“无用”，但实际上它能完美分类（看方差或分布形状）。

  • 分类效用： 适合作为粗筛。如果均值差很大，分类肯定容易；但均值差小，不代表没法分类。

第二类：决策逻辑指标 (Precision, Recall, F1)

  • 统计学视角： 这些不是纯粹的统计量，而是后验评估 (Posterior Evaluation)。它们必须先经过一个非线性的硬阈值操作 (Hard Thresholding/Binarization)：$f(x) = 1 \text{ if } x > \theta \text{ else } 0$。

  • 代数本质： 集合论的计数与比率。

  • 信息处理：

    • 它们抛弃了“激活强度”的信息，只保留“激活与否”的信息（如果你是在二值化后计算）。

    • 非对称性 (Asymmetry): 这是一个关键特性。Precision/Recall 不关心 True Negatives (TN)。在 SAE 场景下（极其稀疏，TN 极大），这是一个优点。因为大多数时候特征不激活（Safe），我们不希望海量的 0 把指标淹没。

  • 分类效用： 非常适合处理稀疏数据。如果你只关心“当这个特征亮起时，是不是 Unsafe”，那么 Precision 是最直接的衡量。

第三类：分布距离指标 (卡方 $\chi^2$)

  • 统计学视角： 假设检验统计量。它衡量的是“观察到的分布”和“理论上的独立分布”之间的欧氏距离（经过方差归一化）。

  • 代数本质： 加权平方和。平方操作使得它对正负偏差一视同仁（不像均值差有方向）。

  • 信息处理： 处理的是频率 (Frequency) 的异常。

  • 分类效用： 强力推荐用于特征选择。 尤其是对于 SAE 这种离散/稀疏特征，卡方检验能非常敏感地捕捉到那些“虽然出现次数不多，但一出现就高度集中在某类”的特征。

第四类：熵与信息量指标 (互信息 MI)

  • 统计学视角： 这是非参数 (Non-parametric) 统计。它不假设数据服从高斯分布，也不假设线性关系。它利用了所有阶的统计矩（因为 $p(x)$ 包含了分布的所有信息）。

  • 代数本质： 对数运算 ($\log$)。这意味着它处理的是比特 (Bits) 或者是数量级，而不是数值本身。

  • 信息处理： 处理的是概率分布的重叠 (Overlap of Distributions)。

    $$I(X;Y) = H(Y) - H(Y|X)$$

    本质上是计算：两个概率分布究竟有多“不一样”。哪怕是非线性的（例如：特征值在中间是 Safe，两头是 Unsafe），互信息也能捕捉到。

  • 分类效用： 理论上的上限。它是最完美的特征选择指标，因为它直接对应贝叶斯误差率的下界（Fano’s Inequality）。如果 MI 为 0，说明这个特征对分类绝对没用。

3. 针对你的 Pipeline (SAE -> Forest) 的具体建议

既然你是做分类 (Safe/Unsafe)，且使用随机森林集成：

  1. 为什么均值差/Steering 指标不够好？

    • 因为随机森林本身是非线性的，它不需要特征和标签线性相关。均值差只能找到线性特征，会漏掉很多 RF 能利用的非线性特征。

  2. 为什么 F1/(P,R) 有风险？

    • 因为它们需要你预先设定一个阈值（比如激活>0）。但这可能不是最优切分点。随机森林的优势就是自己寻找最优切分点。过早二值化会损失“激活强度”里的信息。

  3. 最终推荐：互信息 (MI) 或 卡方 ($\chi^2$)

    • 首选 MI： 因为 SAE 激活值不仅仅是 0/1，其强度大小（Magnitude）往往携带语义浓度。MI 能同时利用“是否激活”和“激活强度”的信息。

    • 次选 $\chi^2$： 如果计算资源受限，或者你打算把 SAE 特征视为纯二值（亮/灭），卡方是最稳健的统计学选择。
