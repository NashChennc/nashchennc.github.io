1. 蒙特卡洛方法 (Monte Carlo Methods)

蒙特卡洛方法利用经验平均值来逼近期望值。其核心思想是：实际进行一整个回合的游戏（作为样本），为每个时刻的状态 St,AtS_t,A_tSt​,At​ 计算回合内的长期回报函数 GtG_tGt​ ，也就是以 GtG_tGt​ 为样本。动作价值函数 QQQ 是长期回报样本 GtG_tGt​ 的期望。$$Q(s, a) \approx \mathbb{E}_{\pi}[G_t \mid S_t=s, A_t=a]$$执行过程 (Episode-wise Update)

  1. 轨迹生成 (Rollout): 利用策略 π\piπ 生成一完整回合 (Episode)：S0,A0,R1,S1,A1,…,ST,RTS_0, A_0, R_1, S_1, A_1, \dots, S_T, R_TS0​,A0​,R1​,S1​,A1​,…,ST​,RT​。
  2. 逆序回溯 (Backward Evaluation): 对幕中出现的每一步 t=T−1,T−2,…,0t = T-1, T-2, \dots, 0t=T−1,T−2,…,0：
    • 计算从 ttt 时刻直至终止状态的累积折现回报 GtG_tGt​。$$G_t \doteq R_{t+1}+\gamma R_{t+2}+\gamma^2 R_{t+3}+\cdots=\sum_{k=0}^{\infty} \gamma^k R_{t+k+1}$$
    • 更新法则: 若采用增量式均值更新 (Incremental Implementation)，则执行：$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[ G_t - Q(S_t, A_t) \right]$$
  3. 重复直至 Q(s,a)Q(s,a)Q(s,a) 收敛

两种可选的模式

  • First-visit: 一局里如果你绕圈回到了同一个状态，只算第一次经过时的回报（无偏估计）。
  • Every-visit: 每次经过都算（在某些情况下收敛更快）。

2. 时序差分控制：SARSA (On-policy TD Control)

SARSA 边走边学，并且怎么走就怎么学。如果不小心走了一步烂棋（探索），就降低对当前状态的评价。这是一种同策略 (On-policy) 算法，即将实际生成的策略直接用于更新价值。

在每一时间步中，使用当前行为策略 π\piπ 的动作进行决策 At+1A_{t+1}At+1​ ，并使用当前决策的 Q(St+1,At+1)Q(S_{t+1}, A_{t+1})Q(St+1​,At+1​) 与上一状态 Q(St,At)Q(S_t, A_t)Q(St​,At​) 之差得到更新方向。这是时序差分。

其名称源于更新元组 (St,At,Rt+1,St+1,At+1)(S_t, A_t, R_{t+1}, S_{t+1}, A_{t+1})(St​,At​,Rt+1​,St+1​,At+1​)。

执行过程 (Step-wise Update) 在每一个时间步 (Step) 中：

  1. 动作选择 (On-policy): 观察由旧状态 (St,At)(S_t, A_t)(St​,At​) 生成的奖励和新状态 (Rt+1,St+1)(R_{t+1},S_{t+1})(Rt+1​,St+1​)。在新状态 St+1S_{t+1}St+1​ 下，依据当前策略 π\piπ 选择下一个动作 At+1A_{t+1}At+1​。这个动作将被最终执行。
  2. 价值更新:
    使用当前决策的 Q(St+1,At+1)Q(S_{t+1}, A_{t+1})Q(St+1​,At+1​) 与旧状态 Q(St,At)Q(S_t, A_t)Q(St​,At​) 之差，使用五元组 (St,At,Rt+1,St+1,At+1)(S_t, A_t, R_{t+1}, S_{t+1}, A_{t+1})(St​,At​,Rt+1​,St+1​,At+1​) 更新 QQQ 值：$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[ \underbrace{R_{t+1} + \gamma Q(S_{t+1}, A_{t+1})}_{\text{TD Target}} - Q(S_t, A_t) \right]$$
  3. 状态推进: 照常执行已选定的动作 At+1A_{t+1}At+1​ ，令 S←St+1,A←At+1S \leftarrow S_{t+1}, A \leftarrow A_{t+1}S←St+1​,A←At+1​。

对应使用估计-迭代法解一个 Bellman 方程 $$Q(S, A) = R’(S,A) + \gamma Q(S’(S,A), A’(S’\mid \pi))$$

3. Q学习 (Q-learning)

Q-learning 是一种异策略 (Off-policy) 算法。无论行为策略如何产生数据，该算法均试图直接逼近最优动作价值函数 Q∗Q_*Q∗​。

执行过程 (Step-wise Update) 在每一个时间步 (Step) 中：

  1. 行为动作选择: 在状态 SSS 下，根据行为策略 (如 ϵ\epsilonϵ-greedy) 选择动作 AAA。
  2. 环境响应: 执行动作 AAA，观测由环境生成的奖励 RRR 和新状态 S′S'S′。
  3. 目标值构造 (Greedy): 不考虑下一步实际将采取的动作，而是寻找 S′S'S′ 下使 QQQ 值最大的动作 aaa 来构造目标：$$y = R + \gamma \max_{a \in \mathcal{A}} Q(S’, a)$$
  4. 价值更新:$$Q(S, A) \leftarrow Q(S, A) + \alpha \left[ y - Q(S, A) \right]$$
  5. 状态推进: 令 S←S′S \leftarrow S'S←S′。

3.1. 插入：在策算法与离策算法

在悬崖行走 (Cliff Walking) 任务中，SARSA 和 Q-learning 会学出完全不同的策略。这是强化学习里最经典的对比案例之一。

1. 现象对比

  • Q-learning (大胆派)：通常收敛得快，且发现的路径更短。它学出的路线是紧贴着悬崖边走（最优路径，步数最少）。

    • 后果：虽然理论上是最快的，但在训练过程中，因为有 ϵ\epsilonϵ (epsilon) 的概率会随机乱走，所以它经常会“脚滑”掉进悬崖，导致训练时的平均回报波动很大。

  • SARSA (保守派)：通常收敛得稍微慢一点，且最终平均奖励（Reward）可能稍微低一点（因为它走远路）。它学出的路线通常是离悬崖有一段距离的安全路线（比如往上走一行再过去）。

    • 后果：虽然步数多了一点（每一步 -1 分），但它极少掉进悬崖。

要从数学角度严谨地解释SARSA（On-Policy） 与 Q-learning（Off-Policy） 在风险偏好上的区别，我们需要深入分析它们各自试图逼近的贝尔曼方程（Bellman Equation） 以及在 ϵ\epsilonϵ-greedy 策略下的期望值差异。

核心结论是：Q-learning 是风险中性（Risk-Neutral）甚至乐观的，因为它假设未来遵循最优策略；而 SARSA 是风险规避（Risk-Averse）的，因为它在价值评估中，通过实际的动作显式地计入了探索（Exploration）带来的潜在震荡或灾难性后果。

证明详见文章附录。

5. 双重 Q 学习 (Double Q-learning)

在 Q-learning 中，目标值 yyy 包含最大化操作 max⁡aQ(S′,a)\max_a Q(S', a)maxa​Q(S′,a)。若 QQQ 值包含随机误差，该操作倾向于高估真实价值。Double Q-learning 利用两个独立的估计器解耦选择与评估过程。

Double Q-learning 是为了解决 Q-learning 的一个通病：Maximization Bias (最大化偏差)。简单说就是 Q-learning 有时候会过度自信，高估某些动作的价值。

执行过程 (Step-wise Update) 在每一个时间步 (Step) 中：

  1. 动作选择: 基于 Q1Q_1Q1​ 和 Q2Q_2Q2​ 的聚合 (例如 Qsum=Q1+Q2Q_{sum} = Q_1 + Q_2Qsum​=Q1​+Q2​)，利用策略选择动作 AAA。
  2. 环境响应: 执行 AAA，观测奖励 RRR 和新状态 S′S'S′。
  3. 随机更新: 以 0.50.50.5 的概率决定更新 Q1Q_1Q1​ (反之更新 Q2Q_2Q2​)。若更新 Q1Q_1Q1​：
    • Argmax (选择): 利用 Q1Q_1Q1​ 确定在 S′S'S′ 下的最优动作索引：$$a^* = \operatorname*{argmax}_{a \in \mathcal{A}} Q_1(S’, a)$$
    • Evaluate (评估): 利用 Q2Q_2Q2​ 估计该动作的价值：$$y = R + \gamma Q_2(S’, a^*)$$
    • 更新法则:$$Q_1(S, A) \leftarrow Q_1(S, A) + \alpha [y - Q_1(S, A)]$$
  4. 状态推进: 令 S←S′S \leftarrow S'S←S′。

1. 算法更新规则的数学形式

首先定义状态 sss，动作 aaa，奖励 rrr，折扣因子 γ\gammaγ，学习率 α\alphaα。

Q-learning 的目标是逼近最优动作价值函数 Q∗Q^*Q∗。其更新公式为：

Q(st,at)←Q(st,at)+α[rt+1+γmax⁡a′Q(st+1,a′)−Q(st,at)]Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha \left[ r_{t+1} + \gamma \max_{a'} Q(s_{t+1}, a') - Q(s_t, a_t) \right] Q(st​,at​)←Q(st​,at​)+α[rt+1​+γa′max​Q(st+1​,a′)−Q(st​,at​)]

2. 风险偏好的数学本质：期望值的展开

为了体现风险偏好，我们必须分析在收敛过程中，这两种算法如何评估一个处于“危险边缘”的状态。

假设当前策略 π\piπ 为 ϵ\epsilonϵ-greedy 策略，即：

  • 以 1−ϵ1-\epsilon1−ϵ 的概率选择贪婪动作 a∗=arg⁡max⁡aQ(s,a)a^* = \arg\max_a Q(s, a)a∗=argmaxa​Q(s,a)。

  • 以 ϵ\epsilonϵ 的概率随机选择任意动作（包括最优动作）。

Q-learning 的视角（最大化算子）

Q-learning 的 TD Target 的期望值（对环境转移动态 PPP 取期望）为：

E[YQL∣st=s,at=a]=∑s′P(s′∣s,a)[R(s,a,s′)+γmax⁡a′Q(s′,a′)]\mathbb{E}[Y_{QL} | s_t=s, a_t=a] = \sum_{s'} P(s'|s,a) \left[ R(s,a,s') + \gamma \max_{a'} Q(s', a') \right] E[YQL​∣st​=s,at​=a]=s′∑​P(s′∣s,a)[R(s,a,s′)+γa′max​Q(s′,a′)]

这个公式表明，Q-learning 完全忽略了探索策略 ϵ\epsilonϵ 带来的噪声。它假设：“只要我学好了，未来我一定会走最完美的那条路”。因此，即使某个状态 s′s's′ 旁边就是悬崖（负奖励极大），只要存在一条安全路径（最优动作），Q-learning 就会给 s′s's′ 极高的估值。

SARSA 的视角（期望算子）

SARSA 的 TD Target 的期望值不仅要对环境转移 PPP 取期望，还要对下一步的策略 π\piπ 取期望：

E[YSARSA∣st=s,at=a]=∑s′P(s′∣s,a)[R(s,a,s′)+γ∑a′π(a′∣s′)Q(s′,a′)]\mathbb{E}[Y_{SARSA} | s_t=s, a_t=a] = \sum_{s'} P(s'|s,a) \left[ R(s,a,s') + \gamma \sum_{a'} \pi(a'|s') Q(s', a') \right] E[YSARSA​∣st​=s,at​=a]=s′∑​P(s′∣s,a)[R(s,a,s′)+γa′∑​π(a′∣s′)Q(s′,a′)]

让我们展开 ∑a′π(a′∣s′)Q(s′,a′)\sum_{a'} \pi(a'|s') Q(s', a')∑a′​π(a′∣s′)Q(s′,a′) 这一项。假设动作空间大小为 ∣A∣|\mathcal{A}|∣A∣，在 ϵ\epsilonϵ-greedy 策略下：

Eπ[Q(s′,⋅)]=(1−ϵ)max⁡a′Q(s′,a′)+ϵ∑a′1∣A∣Q(s′,a′)\mathbb{E}_{\pi}[Q(s', \cdot)] = (1 - \epsilon) \max_{a'} Q(s', a') + \epsilon \sum_{a'} \frac{1}{|\mathcal{A}|} Q(s', a') Eπ​[Q(s′,⋅)]=(1−ϵ)a′max​Q(s′,a′)+ϵa′∑​∣A∣1​Q(s′,a′)

(注：这里采用了简化的 ϵ\epsilonϵ-greedy 定义，有时 ϵ\epsilonϵ 仅分配给非贪婪动作，结论类似)

我们可以将其重写为：

Eπ[Q(s′,⋅)]=max⁡a′Q(s′,a′)⏟Q-learning Term−ϵ(max⁡a′Q(s′,a′)−Qˉ(s′))⏟Risk Penalty (Safety Margin)\mathbb{E}_{\pi}[Q(s', \cdot)] = \underbrace{\max_{a'} Q(s', a')}_{\text{Q-learning Term}} - \underbrace{\epsilon \left( \max_{a'} Q(s', a') - \bar{Q}(s') \right)}_{\text{Risk Penalty (Safety Margin)}} Eπ​[Q(s′,⋅)]=Q-learning Terma′max​Q(s′,a′)​​−Risk Penalty (Safety Margin)ϵ(a′max​Q(s′,a′)−Qˉ​(s′))​​

其中 Qˉ(s′)\bar{Q}(s')Qˉ​(s′) 是所有动作价值的平均值。

3. 结论分析

从上面的公式可以看出 SARSA 与 Q-learning 的本质区别在于那个 Risk Penalty 项：

  1. 风险来源：如果状态 s′s's′ 处于悬崖边缘，那么它的最优动作 Q(s′,asafe)Q(s', a_{safe})Q(s′,asafe​) 可能很高，但错误动作 Q(s′,acliff)Q(s', a_{cliff})Q(s′,acliff​) 会极低（例如 -100）。

  2. Q-learning (Risk-Seeking)：只看 max⁡Q\max QmaxQ。它认为 V(s′)≈Q(s′,asafe)V(s') \approx Q(s', a_{safe})V(s′)≈Q(s′,asafe​)。它不关心如果你手滑（ϵ\epsilonϵ 概率）掉下去会发生什么。因此，它会贴着悬崖边走最优路径。

  3. SARSA (Risk-Averse)：看 E[Q]\mathbb{E}[Q]E[Q]。由于 ϵ\epsilonϵ 的存在，Q(s′,acliff)Q(s', a_{cliff})Q(s′,acliff​) 的极低值会拉低整体期望。

    • 数学上：Qˉ(s′)\bar{Q}(s')Qˉ​(s′) 远小于 max⁡Q(s′,a′)\max Q(s', a')maxQ(s′,a′)。

    • 结果：Risk Penalty 很大，导致 SARSA 降低对 s′s's′ 的估值。

    • 行为：SARSA 会认为 s′s's′ 是危险的，从而倾向于选择离悬崖更远的安全路径，即使那条路稍远一点（次优）。

4. 总结：收敛性与应用

  • Q-learning 收敛于 Q∗Q^*Q∗ (Optimal Value Function)。它学习的是“理论上的最优解”，假设推断时没有探索噪声。

  • SARSA 收敛于 QπQ^{\pi}Qπ (Value Function of the exploration policy)。它学习的是“在当前充满随机性的策略下的实际价值”。

只要 ϵ>0\epsilon > 0ϵ>0，SARSA 就会比 Q-learning 更保守。只有当 ϵ→0\epsilon \to 0ϵ→0（即 GLIE 条件满足，Greedy in the Limit with Infinite Exploration）时，SARSA 的收敛结果才会趋近于 Q-learning。

-2. 最大化偏差

使用数学语言严谨地解释 Double Q-learning 为什么有效，我们需要聚焦于 Q-learning 的一个核心数学缺陷：最大化偏差（Maximization Bias）。

Double Q-learning 的核心贡献在于通过**解耦（Decoupling）**动作的选择与动作的评估，消除了标准 Q-learning 因噪声引起的高估（Overestimation）现象。

以下是严谨的数学推导：

1. 问题的根源：单一估计量的最大化偏差

在标准的 Q-learning 中，目标值（Target）的计算公式为：

YQ=r+γmax⁡a′Q(s′,a′)Y_{Q} = r + \gamma \max_{a'} Q(s', a') YQ​=r+γa′max​Q(s′,a′)

噪声模型

假设对于某个状态 s′s's′，真实的动作价值为 Q∗(s′,a)Q^*(s', a)Q∗(s′,a)。由于函数近似误差或环境随机性，我们的估计量 Q(s′,a)Q(s', a)Q(s′,a) 包含噪声：

Q(s′,a)=Q∗(s′,a)+ϵaQ(s', a) = Q^*(s', a) + \epsilon_a Q(s′,a)=Q∗(s′,a)+ϵa​

其中 ϵa\epsilon_aϵa​ 是均值为 0 的随机误差变量。

期望不等式（Jensen 不等式的变体）

在数学上，对于一组随机变量 X1,X2,…,XnX_1, X_2, \dots, X_nX1​,X2​,…,Xn​，期望的最大值不等于最大值的期望，且通常前者更小。即：

E[max⁡aQ(s′,a)]≥max⁡aE[Q(s′,a)]=max⁡aQ∗(s′,a)\mathbb{E} [ \max_a Q(s', a) ] \ge \max_a \mathbb{E} [ Q(s', a) ] = \max_a Q^*(s', a) E[amax​Q(s′,a)]≥amax​E[Q(s′,a)]=amax​Q∗(s′,a)

直观解释：

假设所有动作的真实价值都为 0（Q∗(s′,a)=0Q^*(s', a)=0Q∗(s′,a)=0），但由于采样噪声，有的估计值是 0.10.10.1，有的是 −0.1-0.1−0.1。

  • max⁡aQ(s′,a)\max_a Q(s', a)maxa​Q(s′,a) 会倾向于选中那个正向误差最大的动作（0.10.10.1）。

  • 因此，更新目标 YQY_QYQ​ 会持续地大于真实值。

  • 随着迭代进行，这种**高估（Overestimation）**会在自举（Bootstrapping）过程中不断累积，导致策略次优甚至震荡。

2. 解决方案：Double Q-learning 的数学形式

Double Q-learning 引入了两个独立的价值函数估计器 Q1Q_1Q1​ 和 Q2Q_2Q2​（通常参数分别为 θ1,θ2\theta_1, \theta_2θ1​,θ2​）。

在更新 Q1Q_1Q1​ 时，其 TD Target 计算如下：

YDQ=r+γQ2(s′,arg⁡max⁡a′Q1(s′,a′)⏟Action Selection)Y_{DQ} = r + \gamma Q_2( s', \underbrace{\arg\max_{a'} Q_1(s', a')}_{\text{Action Selection}} ) YDQ​=r+γQ2​(s′,Action Selectionarga′max​Q1​(s′,a′)​​)

这里发生了关键的解耦：

  1. 动作选择（Selection）：使用 Q1Q_1Q1​ 来决定哪个动作是“最好”的。令 a∗=arg⁡max⁡a′Q1(s′,a′)a^* = \arg\max_{a'} Q_1(s', a')a∗=argmaxa′​Q1​(s′,a′)。

  2. 动作评估（Evaluation）：使用 Q2Q_2Q2​ 来计算这个被选中的动作 a∗a^*a∗ 的价值。

3. 为什么解耦能消除偏差？

我们需要证明 E[YDQ]\mathbb{E}[Y_{DQ}]E[YDQ​] 不存在正向偏差。

假设 Q1Q_1Q1​ 和 Q2Q_2Q2​ 的噪声是相互独立的（在深度学习中通过使用 Target Network 滞后更新来近似实现这一点）。

推导

我们考察 Q2(s′,a∗)Q_2(s', a^*)Q2​(s′,a∗) 的期望，其中 a∗a^*a∗ 是由 Q1Q_1Q1​ 决定的：

E[Q2(s′,a∗)]=E[Q∗(s′,a∗)+ϵ2,a∗]\mathbb{E}[ Q_2(s', a^*) ] = \mathbb{E}[ Q^*(s', a^*) + \epsilon_{2, a^*} ] E[Q2​(s′,a∗)]=E[Q∗(s′,a∗)+ϵ2,a∗​]

由于 Q2Q_2Q2​ 的噪声 ϵ2\epsilon_2ϵ2​ 是均值为 0 的，且与 Q1Q_1Q1​（即 a∗a^*a∗ 的选择过程）独立，因此：

E[Q2(s′,a∗)]=Q∗(s′,a∗)\mathbb{E}[ Q_2(s', a^*) ] = Q^*(s', a^*) E[Q2​(s′,a∗)]=Q∗(s′,a∗)

关键对比

  • Single Q-learning: 即使 a∗a^*a∗ 选错了（因为 Q1Q_1Q1​ 的噪声 ϵ1,a∗\epsilon_{1, a^*}ϵ1,a∗​ 很大），我们也直接使用了这个包含大噪声的值 Q1(s′,a∗)Q_1(s', a^*)Q1​(s′,a∗) 来更新，导致正向偏差。

  • Double Q-learning: 如果 Q1Q_1Q1​ 因为正向噪声 ϵ1\epsilon_{1}ϵ1​ 很大而错误地认为 a∗a^*a∗ 是最优动作，我们转而询问 Q2Q_2Q2​ 对 a∗a^*a∗ 的看法。由于 Q2Q_2Q2​ 的噪声与 Q1Q_1Q1​ 独立，Q2(s′,a∗)Q_2(s', a^*)Q2​(s′,a∗) 大概率不会同时也包含巨大的正向噪声。实际上，它是一个无偏估计。

因此，Double Q-learning 的估计近似满足：

E[Q2(s′,arg⁡max⁡a′Q1(s′,a′))]≤max⁡aQ∗(s′,a)\mathbb{E} [ Q_2(s', \arg\max_{a'} Q_1(s', a')) ] \le \max_a Q^*(s', a) E[Q2​(s′,arga′max​Q1​(s′,a′))]≤amax​Q∗(s′,a)

这就消除了系统性的高估误差。实际上，Double Q-learning 有时甚至会产生轻微的低估（Underestimation），但在强化学习中，低估通常比高估更稳定，因为它不会导致智能体盲目自信地陷入局部极值。

4. 总结

从数学结构上看，Double Q-learning 将算子 max⁡aQ(a)\max_a Q(a)maxa​Q(a) 拆解为 QB(arg⁡max⁡aQA(a))Q_B(\arg\max_a Q_A(a))QB​(argmaxa​QA​(a))。

  • Q-learning：E[max⁡(⋅)]\mathbb{E}[\max(\cdot)]E[max(⋅)]，这是一个凸函数，导致正向偏差。

  • Double Q-learning：利用两个独立统计量，使得选择过程产生的偏差无法传递到评估过程中。

这就像在很多学术会议的审稿机制中：如果作者（Q1Q_1Q1​）自己声称这篇论文是最好的（max⁡\maxmax），往往有偏差；但如果让作者选出他认为最好的论文，然后由独立的评审人（Q2Q_2Q2​）来打分，结果通常更客观。
