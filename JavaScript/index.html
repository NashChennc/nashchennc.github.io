如果你的日志报错涉及 leveldb 和 electron，这些通常包含 C++ 编写的原生模块。如果 Node 版本变了，或者环境不一致，需要重新编译。

执行完方案一的重装后。 强制重新编译原生模块：

  • 删除 pnpm-lock.yaml：确保 pnpm 重新解析依赖树，获取 package.json 中允许的最新版本（有时候锁文件里锁定的版本有 Bug）。

  • pnpm rebuild：你的报错里有 IO error ... leveldb，这往往是因为原生模块没有针对当前的 Electron 版本正确编译。rebuild 可以解决这个问题。
