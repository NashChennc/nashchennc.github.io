Auto Mode 在权限模式体系中的位置

Claude Code 定义了一组权限模式（PermissionMode），用户通过 Shift+Tab 在 REPL 中循环切换。auto mode 是在 TRANSCRIPT_CLASSIFIER 特性开启时追加的内部模式。

stateDiagram-v2
    [*] --> modeDefault
    state "default" as modeDefault
    modeDefault --> acceptEdits: Shift+Tab
    acceptEdits --> plan: Shift+Tab
    plan --> bypassPermissions: Shift+Tab (如可用)
    plan --> auto: Shift+Tab (如可用, 无 bypass)
    bypassPermissions --> auto: Shift+Tab (如可用)
    bypassPermissions --> modeDefault: Shift+Tab (无 auto)
    auto --> modeDefault: Shift+Tab
    plan --> modeDefault: Shift+Tab (无 bypass & 无 auto)

轮转逻辑实现在 getNextPermissionMode（职责：基于当前模式与门禁状态计算“下一个可切换模式”）：

轮转到 auto 需要两个条件同时满足：

  • ctx.isAutoModeAvailable

    • 含义：后台异步校验后，系统给你的“可用资格”结果。
    • 来源：由异步 verifyAutoModeGateAccess 写入。（参数 ctx 为工具权限上下文对象，详见 05-gate-circuit-breaker）
    • 特点：偏“权威结论”，但它是异步更新的，不一定每一刻都最新。

  • isAutoModeGateEnabled()

    • 含义：当前这一刻的同步本地检查。
    • 检查项：熔断器状态、settings 是否禁用、当前模型是否支持。
    • 特点：偏“实时闸门”，Shift+Tab 当下就能判断。

实际切换由 cyclePermissionMode 完成（职责：驱动一次模式轮转），它调用 transitionPermissionMode 执行副作用（职责：真正迁移权限状态并更新上下文；src/utils/permissions/getNextPermissionMode.ts L88-101）。

进入 Auto Mode 的副作用

transitionPermissionMode（src/utils/permissions/permissionSetup.ts L597-640）在切换到 auto 时执行两个关键副作用：

  1. setAutoModeActive(true) — 激活运行时标记
  2. stripDangerousPermissionsForAutoMode(context) — 剥离危险的静态 allow 规则（详见 03-permission-stripping）

离开 auto 时执行逆操作：setAutoModeActive(false) + restoreDangerousPermissions(ctx)（恢复进入前暂存规则）+ setNeedsAutoModeExitAttachment(true)（标记需要附加退出说明）。

这张图里 5 个参与者（竖列）是：

  • User：你在终端按键的人
  • cyclePermissionMode：负责“轮转到下一个模式”
  • transitionPermissionMode：真正执行模式切换和副作用
  • autoModeState：存放 auto 状态标记
  • stripDangerousPermissionsForAutoMode：进入 auto 时剥离危险规则

按时间顺序（从上往下）：

  1. User -> Cycle: Shift+Tab

    • 用户触发一次模式轮转。

  2. Cycle -> Cycle: getNextPermissionMode ... = 'auto'

    • 轮转逻辑先算“下一个模式是不是 auto”。

  3. Cycle -> Trans: transitionPermissionMode(fromMode, 'auto', ctx)

    • 如果目标是 auto，就把“当前模式 + 目标模式 + 上下文”交给真正的切换函数处理。
    • fromMode = 当前模式；'auto' = 目标模式；ctx = 当前权限上下文。

  4. Trans -> Trans: isAutoModeGateEnabled()?

    • 切换函数内部先做同步门禁复核（实时闸门）。

  5. alt 门禁未通过 -> throw Error

    • 如果复核失败，就直接报错返回，不进入 auto。
    • 时序图里的 alt 表示“分支情况”。

  6. Trans -> State: setAutoModeActive(true)

    • 门禁通过后，先把会话状态标记为“已激活 auto”。

  7. Trans -> Strip: stripDangerousPermissionsForAutoMode(ctx)

    • 然后剥离危险的静态 allow 规则，避免绕过分类器。

  8. Strip -->> Trans: 返回清理后的 context

    • 剥离函数返回新的上下文（规则已净化）。

  9. Trans -->> Cycle: 返回新 context

    • 切换函数把新上下文交回轮转函数，轮转完成。

首次同意弹窗

当用户首次进入 auto mode 且尚未 opt-in 时，interactiveHelpers.tsx 会拦截并弹出同意弹窗（目的：确保首次使用前完成风险知情）：

弹窗内容（AUTO_MODE_DESCRIPTION）明确告知：

Auto mode lets Claude handle permission prompts automatically — Claude checks each tool call for risky actions and prompt injection before executing. Actions Claude identifies as safe are executed, while actions Claude identifies as risky are blocked and Claude may try a different approach.

接受后写入 userSettings：

  • skipAutoPermissionPrompt: true（标记已同意）
  • 可选 permissions.defaultMode: 'auto'（设为默认模式）

hasAutoModeOptIn 的检查故意排除 projectSettings（src/utils/settings/settings.ts L896-908），防止恶意项目通过 .claude/settings.json 自动绕过同意弹窗。
