SSML
Last Update: 2026/03/04 15:34
笔记

SSML（Speech Synthesis Markup Language，语音合成标记语言）是一种基于 XML 的标记语言，用于让开发者能够更精细地控制文本转语音（TTS）的效果。

简单来说，如果普通文本是告诉语音引擎“读什么”，那么 SSML 就是告诉引擎“怎么读”。它是 W3C 的标准协议，被 Google、Amazon (Alexa/Polly)、Microsoft (Azure) 等主流语音服务广泛支持。

1. 核心作用

SSML 允许你调整语音的以下方面，使其听起来更自然、更像真人：

  • 停顿（Breaks）： 在句子或短语之间插入指定时长的停顿。
  • 发音（Pronunciation）： 指定多音字、生僻字或外来语的具体读音（使用音标）。
  • 韵律（Prosody）： 调整语速、音高（声调）和音量。
  • 解释方式（Say-as）： 告诉引擎如何朗读数字、日期、电话号码或缩写。
  • 强调（Emphasis）： 加重语气。

2. 常用标签详解

所有的 SSML 内容都必须包含在 <speak> 根标签内。

A. 停顿 <break>

用于控制节奏。

XML

B. 调整韵律 <prosody>

这是最常用的标签，用于控制语速（rate）、音高（pitch）和音量（volume）。

XML

C. 指定读法 <say-as>

解决歧义非常关键。例如 “2024” 是读作 “二零二四” 还是 “两千零二十四”。

XML

D. 别名/替换 <sub >

用于朗读缩写或特殊字符的完整含义。

XML

E. 插入音频 <audio>

在语音流中混合背景音效或预录制的音频。

XML

F. 拼音/音素 <phoneme>

当引擎读错多音字或人名时，使用国际音标（IPA）或特定平台的注音格式强制指定发音。

XML

(注：不同厂商对 phoneme 的支持格式可能不同)

SSML 是连接“僵硬的机器朗读”与“富有表现力的语音交互”之间的桥梁。虽然现在许多端到端的 AI 模型（如类似 GPT-4o 的语音模式）已经能自动推断语气，但在开发 IVR（电话语音菜单）、有声读物或特定行业应用时，SSML 依然是确保准确性和一致性的最佳工具。
